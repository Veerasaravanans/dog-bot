▼ Phase 1 Install Arduino IDE & Libraries 6 steps
01

Download Arduino IDE 2.x

Open your web browser and navigate to the official Arduino website:

https://www.arduino.cc/en/software

Click the "DOWNLOAD" button for Windows. The file will be approximately 150 MB.

02

Install Arduino IDE

  1. Locate the downloaded file in your Downloads folder
  2. Double-click the .exe file to begin installation
  3. Click "Yes" when Windows prompts for permission
  4. Accept the license agreement by clicking "I Agree"
  5. Keep all default options selected
  6. Click "Install" and wait for completion (2-3 minutes)
  7. Click "Finish" when installation completes
ℹ Arduino IDE will launch automatically. If Windows Firewall appears, click "Allow access".
03

Add ESP32 board support

  1. In Arduino IDE, click File → Preferences
  2. Find the field labeled "Additional boards manager URLs"
  3. Paste the following URL:
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  1. Click "OK" to save
04

Install ESP32 boards package

  1. Click the Boards Manager icon in the left sidebar (second icon from top)
  2. Search for esp32
  3. Find "esp32 by Espressif Systems"
  4. Click "INSTALL"
  5. Wait for installation (3-5 minutes, ~300 MB download)
⚠ Do not close Arduino IDE during installation. The download is large and may take several minutes.
05

Install PubSubClient library (for MQTT)

  1. Click the Library Manager icon in the left sidebar (third icon from top)
  2. Search for PubSubClient
  3. Find "PubSubClient by Nick O'Leary"
  4. Click "INSTALL"
06

Connect Arduino Uno to computer

  1. Take your Arduino Uno board
  2. Connect it to your computer using a USB A-to-B cable
  3. Wait 5 seconds for Windows to detect the device
  4. Verify the green power LED is ON
ℹ Arduino Uno uses standard drivers that come with Windows and Arduino IDE. No additional driver installation needed.
▼ Phase 2 Wire ESP32-CAM to Arduino Uno for Programming 11 steps
01

Why use Arduino Uno to program ESP32-CAM?

The ESP32-CAM module doesn't have a built-in USB port. We'll use the Arduino Uno as a USB-to-Serial bridge to upload firmware to the ESP32-CAM.

The Arduino Uno will act as a "dumb passthrough" device that relays serial data between your computer and the ESP32-CAM.

02

Gather required materials

  1. ESP32-CAM module
  2. Arduino Uno board (connected to computer)
  3. 6× Female-to-male jumper wires
  4. 1× Male-to-male jumper wire
03

Understanding the wiring diagram

Here's a visual representation of the connections:

⚠ CRITICAL CONNECTIONS: TX→RX and RX→TX (wires are CROSSED!). IO0→GND enables bootloader mode but MUST be removed after upload!
04

Detailed wiring table

# Arduino Uno Pin Wire Type ESP32-CAM Pin Notes
1 RESET Male-to-Male GND (any) DISABLES Arduino CPU - REQUIRED!
2 5V Female-to-Male 5V Power for ESP32-CAM
3 GND (any) Female-to-Male GND (bottom) Common ground - REQUIRED!
4 TX (Pin 1) Female-to-Male U0R (RX) CROSSED! TX→RX
5 RX (Pin 0) Female-to-Male U0T (TX) CROSSED! RX→TX
6 GND (any) Female-to-Male IO0 BOOTLOADER MODE - Remove after upload!
05

Connection #1: Disable Arduino Uno's processor (RESET→GND)

  1. Take a male-to-male jumper wire
  2. Connect one end to Arduino Uno's RESET pin
  3. Connect the other end to any GND pin on Arduino Uno
⚠ This wire MUST stay connected during upload. The Arduino's built-in LED (pin 13) should turn OFF, confirming the CPU is disabled.
06

Connection #2: Power - 5V to 5V

  1. Take a female-to-male jumper wire (RED recommended)
  2. Plug male end into Arduino Uno's 5V pin
  3. Plug female end onto ESP32-CAM's 5V pin
07

Connection #3: Ground - GND to GND

  1. Take a female-to-male jumper wire (BLACK recommended)
  2. Plug male end into Arduino Uno's GND pin
  3. Plug female end onto ESP32-CAM's GND pin (bottom pins, leftmost)
⚠ Common ground is MANDATORY for serial communication to work!
08

Connection #4: Serial Data - TX to RX (CROSSED!)

  1. Take a female-to-male jumper wire (BLUE or GREEN recommended)
  2. Plug male end into Arduino Uno's TX (Pin 1)
  3. Plug female end onto ESP32-CAM's U0R (RX) pin
⚠ TX goes to RX (transmit to receive). The wires are CROSSED!
09

Connection #5: Serial Data - RX to TX (CROSSED!)

  1. Take a female-to-male jumper wire (ORANGE or YELLOW recommended)
  2. Plug male end into Arduino Uno's RX (Pin 0)
  3. Plug female end onto ESP32-CAM's U0T (TX) pin
10

Connection #6: Bootloader Mode - GND to IO0

  1. Take a female-to-male jumper wire (WHITE recommended)
  2. Plug male end into another Arduino Uno GND pin
  3. Plug female end onto ESP32-CAM's IO0 pin
⚠ This wire MUST be connected DURING upload, but MUST be REMOVED AFTER upload completes!
11

Verify all connections

  1. ✓ Arduino RESET → Arduino GND (male-to-male)
  2. ✓ Arduino 5V → ESP32-CAM 5V
  3. ✓ Arduino GND → ESP32-CAM GND
  4. ✓ Arduino TX → ESP32-CAM U0R (CROSSED)
  5. ✓ Arduino RX → ESP32-CAM U0T (CROSSED)
  6. ✓ Arduino GND → ESP32-CAM IO0
ℹ Total wires: 6. ESP32-CAM's red power LED should be ON if connections are correct.
▼ Phase 3 Configure & Upload Firmware to ESP32-CAM 14 steps
01

Download the DogBot firmware

  1. Navigate to your project folder: C:\Users\vese300082\dog-bot\code
  2. Open the firmware folder: firmware\esp32cam_dogbot\
  3. Locate the file: esp32cam_dogbot.ino
02

Open firmware in Arduino IDE

  1. Right-click esp32cam_dogbot.ino
  2. Select "Open with" → "Arduino IDE"
  3. Arduino IDE will open with the firmware loaded
03

Configure WiFi credentials

Locate lines 14-15 in the code and replace with your WiFi network name and password:

const char* ssid = "YOUR_WIFI_NAME"; const char* password = "YOUR_WIFI_PASSWORD";
⚠ Use 2.4GHz WiFi network only. ESP32-CAM does NOT support 5GHz WiFi!
04

Configure MQTT broker credentials (optional)

If you want remote control via MQTT, update lines 20-26 with your HiveMQ Cloud credentials:

const char* mqtt_broker = "your-cluster.hivemq.cloud"; const int mqtt_port = 8883; const char* mqtt_username = "your_username"; const char* mqtt_password = "your_password"; const char* mqtt_topic_command = "dogbot/command"; const char* mqtt_topic_status = "dogbot/status";
ℹ If you don't have MQTT broker, leave these as-is. Local HTTP control will still work.
05

Select the correct board

  1. Click Tools → Board → ESP32 Arduino
  2. Scroll down and select "AI Thinker ESP32-CAM"
⚠ MUST select "AI Thinker ESP32-CAM" - other ESP32 boards won't work correctly!
06

Select the COM port

  1. Click Tools → Port
  2. Select the COM port with "Arduino Uno" in parentheses
  3. Example: COM3 (Arduino Uno)
ℹ If you don't see any ports, reconnect the Arduino Uno USB cable and wait 5 seconds.
07

Configure upload settings

Set these exact settings in the Tools menu:

  1. Upload Speed: 115200
  2. Flash Frequency: 80MHz
  3. Flash Mode: QIO
  4. Partition Scheme: Huge APP (3MB No OTA/1MB SPIFFS)
08

Verify all wiring is still connected

  1. ✓ Arduino RESET → Arduino GND (wire #1)
  2. ✓ Arduino 5V → ESP32-CAM 5V (wire #2)
  3. ✓ Arduino GND → ESP32-CAM GND (wire #3)
  4. ✓ Arduino TX → ESP32-CAM U0R (wire #4)
  5. ✓ Arduino RX → ESP32-CAM U0T (wire #5)
  6. ✓ Arduino GND → ESP32-CAM IO0 (wire #6) - MUST be connected!
⚠ Wire #6 (GND→IO0) MUST be connected for upload to work. This enables bootloader mode.
09

Press the RST button on ESP32-CAM

  1. Locate the small black button on ESP32-CAM labeled "RST"
  2. Press and release it once
  3. This resets the ESP32-CAM into bootloader mode
ℹ If your ESP32-CAM doesn't have a reset button, briefly disconnect and reconnect the 5V power wire instead.
10

Upload the firmware

  1. Click the Upload button (right arrow icon) in Arduino IDE
  2. Wait for compilation (30-60 seconds)
  3. Watch the output window for upload progress
  4. Upload takes 2-3 minutes at 115200 baud
⚠ Do NOT disconnect any wires during upload! If upload fails with "A fatal error occurred: Failed to connect", press RST button again and retry upload immediately.
11

Wait for upload completion

You should see this output sequence:

Connecting........_____....._____..... Writing at 0x00010000... (10%) Writing at 0x00020000... (20%) ... Writing at 0x000e0000... (100%) Wrote 1234567 bytes (654321 compressed) at 0x00010000 in 45.6 seconds Hash of data verified. Leaving... Hard resetting via RTS pin...
ℹ "Hard resetting via RTS pin..." means upload succeeded!
12

CRITICAL: Remove the IO0 bootloader wire

  1. Disconnect wire #6 (GND → IO0 orange wire) from ESP32-CAM
  2. Leave all other 5 wires connected
⚠ You MUST remove this wire! If left connected, ESP32 will stay in bootloader mode and won't run your program!
13

Open Serial Monitor to verify

  1. Click Tools → Serial Monitor
  2. Set baud rate to 115200 (bottom-right dropdown)
  3. Press the RST button on ESP32-CAM
14

Verify serial output

You should see this output in Serial Monitor:

=== DogBot ESP32-CAM Starting === Camera init: OK L298N with PWM speed control Connecting to WiFi: YOUR_WIFI_NAME WiFi connected: 192.168.1.100 Camera stream: http://192.168.1.100:81/stream Control server started on port 80 MQTT connecting to broker... MQTT connected successfully!

Key things to verify:

  • ✓ WiFi connected with IP address shown
  • ✓ Camera stream URL displayed
  • ✓ "L298N with PWM speed control" message
  • ✓ If using MQTT: "MQTT connected successfully"
ℹ Write down the IP address! You'll need it to configure the backend. Example: 192.168.1.100
▼ Phase 4 Setup Cloudflare Tunnel for Remote Access 5 steps
01

Why Cloudflare Tunnel?

The ESP32-CAM video stream (http://192.168.1.100:81/stream) only works on your local WiFi network. To access it remotely from anywhere in the world, we'll use Cloudflare Tunnel to create a secure connection.

Installation steps:

  • go to this webpage and download the winget "https://learn.microsoft.com/en-us/windows/package-manager/winget/"
  • after the page is opened scroll down you can see -- Install WinGet preview version [Developers Only]
  • in that click -- WinGet preview version and download that into your system
  • it is offical windows tool only so do be scared
  • No account creation required
02

Install Cloudflare Tunnel using winget

  1. Open Command Prompt or PowerShell
  2. Press Windows Key + R, type cmd, press Enter
  3. Run this command:
winget install cloudflare.cloudflared
  1. Wait for installation to complete (30 seconds to 1 minute)
  2. You'll see "Successfully installed" when done
  3. Close and reopen Command Prompt to refresh PATH
03

Start the tunnel and get your public URL

  1. Make sure your ESP32-CAM is powered on and connected to WiFi
  2. Note your ESP32-CAM IP address from Phase 3 (e.g., 192.168.1.100)
  3. In Command Prompt, run this command (replace IP address with yours):
cloudflared tunnel --url http://192.168.1.100:81
  1. Wait 5-10 seconds for the tunnel to connect
04

Copy your public URL

You'll see output like this:

+--------------------------------------------------------------------------------------------+ | Your quick Tunnel has been created! Visit it at (it may take some time to be reachable): | | https://random-words-here.trycloudflare.com | +--------------------------------------------------------------------------------------------+
  1. Copy the https://random-words-here.trycloudflare.com URL
  2. This is your public video stream URL!
  3. Write it down - you'll need it in the next step
⚠ IMPORTANT: Keep this Command Prompt window open! Closing it will stop the tunnel. The URL will change each time you restart the tunnel.
ℹ Test it: Open the URL in your browser and add /stream at the end. Example: https://random-words-here.trycloudflare.com/stream - you should see live video!
▼ Phase 5 Final Hardware Assembly with L298N Motor Driver 10 steps
01

Disconnect ESP32-CAM from Arduino Uno

  1. Gently remove all 5 wires connecting ESP32-CAM to Arduino Uno
  2. You can also disconnect Arduino Uno from computer (USB cable)
  3. Set Arduino Uno aside - it's no longer needed
ℹ The firmware is now permanently stored in ESP32-CAM's flash memory. Arduino Uno was only needed for uploading.
02

Gather L298N hardware components

  • L298N motor driver module (red board with heat sink)
  • ESP32-CAM module (with firmware already uploaded)
  • 2× DC motors (for robot movement)
  • 7.4V - 12V battery pack (for motor power)
  • Female-to-female jumper wires (11 wires needed)
03

Understanding the L298N pinout

The L298N has the following key pins:

Pin Purpose
12V Motor power input (7.4V - 12V battery)
GND Common ground
5V Regulated 5V output (can power ESP32-CAM)
ENA Enable Motor A + PWM speed control
IN1 Motor A direction control 1
IN2 Motor A direction control 2
IN3 Motor B direction control 1
IN4 Motor B direction control 2
ENB Enable Motor B + PWM speed control
OUT1/OUT2 Motor A output terminals
OUT3/OUT4 Motor B output terminals
04

L298N to ESP32-CAM wiring reference

Use this pin diagram as reference:

📌 Open Full Pin Diagram
L298N Pin Wire Color ESP32-CAM Pin Function
12V Red — Connect to battery + terminal
GND Black GND Common ground (also to battery -)
5V Red 5V Power for ESP32-CAM
ENA Orange GPIO2 Motor A speed (PWM)
IN1 Yellow GPIO12 Motor A direction 1
IN2 Green GPIO13 Motor A direction 2
IN3 Blue GPIO14 Motor B direction 1
IN4 Purple GPIO15 Motor B direction 2
ENB White GPIO4 Motor B speed (PWM)
05

Connect power wires

  1. Connect battery positive (+) to L298N 12V terminal (use red wire or screw terminal)
  2. Connect battery negative (-) to L298N GND terminal (black wire)
  3. Connect L298N 5V to ESP32-CAM 5V pin (red jumper wire)
  4. Connect L298N GND to ESP32-CAM GND pin (black jumper wire)
⚠ VERIFY battery polarity! Reversed polarity will destroy the L298N and ESP32-CAM!
06

Connect PWM speed control pins

  1. Remove the jumper cap from L298N ENA pin (if present)
  2. Remove the jumper cap from L298N ENB pin (if present)
  3. Connect L298N ENA → ESP32-CAM GPIO2 (orange wire)
  4. Connect L298N ENB → ESP32-CAM GPIO4 (white wire)
⚠ The jumper caps MUST be removed for PWM speed control to work!
07

Connect Motor A direction control

  1. Connect L298N IN1 → ESP32-CAM GPIO12 (yellow wire)
  2. Connect L298N IN2 → ESP32-CAM GPIO13 (green wire)
08

Connect Motor B direction control

  1. Connect L298N IN3 → ESP32-CAM GPIO14 (blue wire)
  2. Connect L298N IN4 → ESP32-CAM GPIO15 (purple wire)
09

Connect the motors

  1. Connect left motor wires to L298N OUT1 and OUT2
  2. Connect right motor wires to L298N OUT3 and OUT4
  3. Motor polarity determines rotation direction (you can swap if motors run backwards)
ℹ If a motor spins the wrong direction during testing, simply swap its two wires at the L298N output terminals.
10

Final verification and power-on test

  1. Verify all 11 connections are secure
  2. Verify battery polarity is correct
  3. Turn on battery power
  4. ESP32-CAM red power LED should light up
  5. Wait 10 seconds for ESP32 to connect to WiFi
ℹ SUCCESS! Your DogBot hardware is now fully assembled. Start the backend server (python run.py) and access the dashboard at http://localhost:8000 to control your robot!
⚠ First-time safety tip: Place robot on blocks so wheels are off the ground when testing motor commands!
